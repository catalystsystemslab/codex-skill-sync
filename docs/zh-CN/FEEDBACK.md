# 反馈指南

Skill Sync 目前处于公开 Beta。最有价值的反馈是关于信任、安全和初学者清晰度。

## 最有用的反馈

请告诉我们：

- 安装说明是否清楚？
- `/skill-sync doctor` 是否容易理解？
- `/skill-sync check` 是否让你信任这个工具？
- Needs Setup / 需要设置 是否让你困惑？
- 在更新前，你是否理解会发生什么？
- 备份和恢复说明是否足够？
- 是否有输出太技术化？
- 是否有失败信息让人害怕或不清楚？

## 应该包含什么

如果可以，请包含：

- 你运行的命令或提示词
- 你看到的输出
- 你使用的 agent/editor：Codex、Claude Code、Cursor 或其他
- 你的操作系统
- 是否安装了 Codex CLI
- 该技能是公开、私有，还是仅本地使用

发帖前，请删除私有仓库 URL、本地用户名、token 或私有路径。

## 当前功能优先级

下一个功能应该来自真实用户遇到的摩擦。

可能的候选：

- 更容易的来源映射
- 恢复命令
- 应用前的文件级 diff 摘要
- 更好的定时检查自动化
- 更简单的安装流程

## 在用户明确需要前暂不计划

这些功能目前有意延后：

- `--map-skill`
- `--restore`
- 文件级 diff 摘要
- 自动定时 apply
- package registry 支持
- 广义 marketplace 行为

Skill Sync 目前定位为安全维护工具，不是通用包管理器。
